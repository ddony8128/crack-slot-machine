# RULE SLOT — UI 디테일과 의도

게임 콘텐츠/점수의 SoT는 [`SPEC.md`](SPEC.md)다. 이 문서는 **화면·문구·연출**에 대한 결정과
**그 이유(의도)**를 남긴다. 대부분 Season 1 QA 라운드에서 실제 플레이 중 발견한 문제를 고치며 정해졌고,
"왜 이렇게 했는가"가 코드만 봐서는 드러나지 않기 때문에 여기 기록한다. 코드와 충돌하면 코드가 현실이고,
의도가 깨졌다면 코드를 고친다.

> 관련 컴포넌트는 각 절에 명시한다. 공통 원칙: **연출 → 결과**. 어떤 모드든 "무슨 일이 일어났는지"를
> 먼저 보여준 뒤에 판정/정산을 띄운다. 기준만 넘기면 슬롯 결과도 없이 "클리어!"가 떠버리는 것이
> QA에서 가장 반복된 불만이었다.

---

## 1. 공통 원칙

1. **연출이 결과보다 먼저다.** 스핀 릴 → 규칙 적용 애니메이션 → 정산/판정 순서를 절대 건너뛰지 않는다.
   클리어/실패가 확정돼도 그 스핀의 reveal을 끝까지 보여준 뒤 결과 화면으로 넘어간다.
   (스토어가 `'spin-result'` 단계를 유지하고 `puzzleCleared`/`stageCleared` 플래그만 세팅 →
   `next()`에서 비로소 종료. replay-safe: 서버가 클리어 여부를 점수로 재계산.)
2. **모드마다 필요한 도움말만 보여준다.** 빠른 게임=규칙/점수표, 첨탑=보유 현황, 퍼즐=심볼 풀.
   불필요한 패널(퍼즐의 점수표, 첨탑의 구식 규칙 버튼)은 제거한다.
3. **문구는 한다체로 통일**하고, 점수·돈·확률 등 숫자는 그 의미를 괄호로 분명히 한다
   (예: "+50 (1원)" — 50점을 1원에 산다는 뜻).
4. **시즌 점수에 어떻게 반영되는지**를 결과 화면에서 항상 설명한다. 플레이어가 "이 140점이 시즌에서
   뭔데?"를 묻지 않게 한다.

---

## 2. 퍼즐 모드

**컴포넌트**: `PuzzlePanel`, `SymbolPool`, `PuzzleResultScreen`, `GameScreen`(`isPuzzle` 분기),
`StatusBar`(`hideReference`).

- **점수표·규칙 도움말 제거 → 심볼 풀 표시.** 퍼즐은 정해진 규칙으로 정해진 목표를 맞추는 모드라
  포커식 점수표가 의미 없다. 대신 어떤 심볼이 나오는지가 핵심이라 `SymbolPool`을 보여준다.
  - **인트로**(퍼즐 진입 시)와 **인게임**(`StatusBar`) 양쪽에서 심볼 풀을 노출한다.
- **스핀별 점수 정산 제거.** 퍼즐은 점수 경쟁이 아니라 클리어 판정이다. `PuzzlePanel`은
  현재 보드가 **클리어 조합인지 아닌지만** 표시한다(점수 숫자를 띄우지 않는다).
- **클리어 연출.** 슬롯 결과가 나오기도 전에 "클리어!"가 떠버리던 버그를 고쳤다(`puzzleCleared` 플래그로
  reveal을 먼저 재생). 
- **결과 화면 문구(`PuzzleResultScreen`).**
  - "달성 목표 N/N"은 목표가 2개 이상일 때만 표시한다(1개짜리 퍼즐에서 "1/1"은 혼란만 줬다).
  - "시즌 점수 획득: +N점"을 명시하고, 공식(클리어 100 + 남은 스핀×10, 퍼즐별 베스트)을 한 줄로 덧붙인다.

## 3. 첨탑 오르기 (Spire)

**컴포넌트**: `SpireClient`, `SpireShop`, `SpireResultScreen`, `HoldingsModal`, `StatusBar`(`isSpire`).

### 3.1 스테이지 클리어/실패 — reveal 후 정산 화면

- 기준 점수만 넘기면 무슨 일이 벌어졌는지 모른 채 "스테이지 1 클리어!"가 떠버리던 문제를 고쳤다.
  이제 **그 스핀의 reveal을 먼저 보여주고**, 이어서 **정산 화면**을 띄운다(`'cleared'`/`'failed'` phase).
- **정산 화면 위치.** 정산 내역(이자 / 남은 스핀 보너스 / 클리어 보수 / 실패 지원금)은 상점 UI 위에
  얹지 않고, **각 스테이지를 끝내며 정산 화면에서 함께** 보여준다. 확인 후 `proceedAfterClear`로 진행.
  - 예: "정산 — 이자 +0 · 남은 스핀 +2 · 클리어 보수 +4".

### 3.2 보유 현황 패널 (구 규칙/점수표 버튼 대체)

- 첨탑의 심볼 주머니·규칙 풀 UI가 빈약했다. 기존 규칙 버튼을 **"보유 현황"** 버튼으로 바꿔
  `HoldingsModal`을 연다. 내용:
  - **심볼 주머니**: `SymbolPool`을 `showWeights`로 — 각 심볼의 개수와 확률 가중치 `×N (%)`를 표시.
    (확률 의문 — "20개 중 1개인데 왜 자주 나와?" — 을 눈으로 확인시키기 위함.)
  - **규칙 풀** + 각 규칙 설명.
  - **아티팩트** + 각 효과 설명.

### 3.3 상점 (`SpireShop`)

- **가격 표기.** 족보 강화를 "+50 (1원)"처럼 **(괄호 안이 구매 금액)**으로 표기해 50점/×2와 가격(1·3원)을
  명확히 구분한다.
- **족보 강화 = 포커 족보 + 보유 세트 족보 둘 다.** 현재 보유한 각 심볼 세트의 보너스도 종류별로 강화 가능.
  양수 보너스는 +50/×2 각각 한 번씩, 페널티(이웃 고양이)는 **완화만** 한 번. 구매 완료한 항목은 "완료"로 표시.
- **품절(no refill).** 구매한 상품은 그 자리에서 **품절**되고 다른 상품으로 채워지지 않는다.
  (그래야 리롤이 의미가 있다.) 상점 진입/리롤 시점의 제안을 스냅샷으로 고정하고, 심볼/가격만 라이브로 합쳐
  소유/풀-포함 항목을 품절 처리한다.
- **리롤 라벨.** 차임벨 보유 시 무료 리롤을 "무료 · 차임벨"로 표기.

### 3.4 진행도 저장 · 이어하기

- "첨탑은 저장되지 않습니다 — 나가면 사라집니다"는 잘못된 안내였다. 실제로는 **각 스테이지 시작·종료 시
  자동 저장**한다. HUD 문구를 "진행 상황은 자동 저장됩니다…"로 바꾸고, 나가기(`exitRun`)는 런을 지우지 않는다.
- 시즌 허브 카드에 "▶ 이어하기 — N스테이지 진행 중"을 띄운다(서버가 도달 스테이지를 재계산).

### 3.5 결과 화면 (`SpireResultScreen`)

- 최종 클리어/실패 후 **시즌 첨탑 랭킹**(`SpireRankingPanel`)을 보여준다 — 다른 사람들의 최종 도달 스테이지와
  점수, 본인 행은 강조. 기록이 등록된 뒤(`submitted`) 불러와 본인 기록도 포함되게 한다.
- 시즌 점수 공식(스테이지×100 + 돈×10 + 남긴 스핀×10)을 한 줄로 명시하고 카운트업 연출(`SeasonScoreRise`).

### 3.6 아티팩트 문구

- 엔진 아티팩트가 "🎁 0 보너스 · 이번 턴 2장 선택"처럼 엉뚱하게 표시되던 문제를 고쳤다.
  규칙 선택 보너스 라벨을 source-neutral하게 바꿔 출처(엔진 등)를 "0 보너스"로 오기하지 않는다.

## 4. 일일 도전

**컴포넌트**: `DailyClient`, `DailyResultScreen`, `DummyAdModal`.

- **특수 족보 숨김.** 일일 도전에는 숫자 특수 족보(4×4→배수 등)를 노출하지 않는다(빠른 게임 전용).
- **재도전 버튼.** 5회(기본 3회 + 충전) 소진 후 결과 화면에 다시 도전 경로가 없던 문제 → 제출 후
  `refreshCurrent`로 남은 횟수를 갱신하고, 소진 시 **광고 충전 → 재도전** 경로를 연다(refill+retry).
- **후원 CTA 타이밍.** 후원 안내가 "10회째 종료 직후"가 아니라 "그 후 다시 접속할 때" 떠서 어색했다.
  → 광고 충전 모달(`DummyAdModal`)에 광고 안내와 함께 **후원 요청**을 넣어, 충전 흐름 안에서 자연스럽게 노출.

## 5. 빠른 게임 / 공통 슬롯 UI

**컴포넌트**: `GameScreen`, `StatusBar`, `ScorePanel`, `ReferenceModal`, `RulePicker`.

- **레거시 동결**: 빠른 게임/이벤트는 원조 룰셋으로 고정한다. 규칙은 id 화이트리스트
  `LEGACY_RULE_IDS`(원조 30개)로, 족보는 원조 색 보너스(올 레드/올 블루)까지 복원해서 노출한다.
  **`규칙 보기`(ReferenceModal)는 실제 오퍼 풀과 동일한 화이트리스트를 쓰므로 "보이는 것 = 나오는 것"**이
  항상 성립한다. 시즌 추가 규칙(비타민 보충·미의 추구)·시즌 세트 규칙은 레거시에 나오지 않는다.
  자세한 경위는 [SPEC §1.1 레거시 동결](SPEC.md#11-빠른-게임) / `docs/STATUS.md`.
- **점수표는 빠른 게임에서만 특수 족보 + 색 보너스를 노출**한다(시즌 모드에선 숨김 — `legacy` 분기).
- **select 계열 규칙 프롬프트.** 직접 고르는 규칙마다 안내 문구를 정확히 매칭한다.
  - 유료 주차: "주차할 교통수단 칸을 선택하세요" (다른 select 규칙 문구가 잘못 나오던 버그 수정).
  - 왜 여기 타 있어(catswap), 물류 사업(logiswap)도 각자 올바른 프롬프트.
- **유료 주차 confirm UI.** 교통수단을 **최대 2칸**까지 토글로 고르고 **확인**으로 확정한다(1칸 또는 2칸 모두 허용).
  강제 2칸도, 무제한도 아니다 — [SPEC Known deviations §1](SPEC.md#known-deviations) 참조.
  - 이전 스핀에 유료 주차로 유지된 칸도 다시 선택 대상에 포함된다.
- **유지(held) 칸 복사.** "유지" 상태인 칸도 SELECT COPY로 복사 대상이 될 수 있다(선택 불가하던 버그 수정).

## 6. 시즌 허브 · 랭킹

**컴포넌트/페이지**: `app/season/page.tsx`, `app/season/leaderboard/page.tsx`.

- **허브 카드**: 첨탑 카드는 진행 중 런이 있으면 "▶ 이어하기 — N스테이지 진행 중"을 앞에 붙인다.
- **랭킹 페이지**: "점수 계산 방법 보기"를 펼치면 첨탑/퍼즐/일일 각 공식을 설명한다(시즌 점수 = 세 모드 합산).
- 일일 도전도 **자체 랭킹**(`/season/daily/leaderboard`)이 있고, 첨탑도 자체 랭킹이 있다. 시즌 통합 랭킹과는 별개.

## 7. 그 외 결정 메모

- **고양이 이동 점수.** 우다다다/점프의 달인은 이동 외에 **고양이 1마리당 +40점**("사라진 고양이 점수")을
  준다 — 점수표/규칙 설명에 반영. (`CAT_MOVE_PER=40`.)
- **확률 가중치는 실제로 적용된다.** 첨탑 심볼 주머니의 `×N`은 표시용이 아니라 실제 가중 샘플링 값이다.
  백귀야행 + COPY ABOVE가 겹치면 가중치가 곱으로 쌓인다(예: 괴물 5 → ×8 × ×8 = ×64). positional weight 규칙이
  `expandRules`로 펼쳐져 COPY ABOVE에 의해 중첩되도록 수정.
- **버전 게이트는 UI가 아니라 신뢰의 경계다.** 결과 화면의 "새 버전이 배포되었어요"(version_mismatch)는
  규칙이 바뀐 뒤 옛 클라이언트로 낸 기록을 조용히 섞지 않기 위함 — 새로고침 후 재플레이를 유도한다.
